Quick formatting rundown:

This is a link. This span blinks ->▏ - and this text is in a div.notice (div with a notice class)

And, this is where you'd usually add a warranty disclaimer.

A good mod should have tool sections (if applicable), sourced parts list (if applicable) with alternate sources (for extra creds), and detailed instructions with images.

But, in all honesty, we know this isn't a viable way to make a mod guide. That's why we're working on guidelines for community mods.

Meanwhile, here's the template:

Tools Required

  • Example Tools
  • If none, we use "Not Applicable"

Materials Used

  • Materials is an umbrella term for additional materials such as random screws you found lying around
  • Or even the type of filament you used in your 3D printer
  • If none, we use "Not Applicable"

Parts Sourcing

  • We all like a mod guide to list the specific part the mod writer used
  • We all like an alternative or two to list cheaper alternatives that are also compatible or that ship to more countries
  • If none, we use "Not Applicable"

Step-by-Step Instructions

  1. This is where you go into as much detail as possible describing the full process of how you did what you did
  2. Feel free to use as many steps as necessary, and add small disclaimers for important things

    3. Like this:

    "Make sure to take pictures of the disassembly process. It makes reassembly easier"

    or:

    "Make sure to write down what steps you performed in the case you want to undo this mod"
